# Управление библиотекой книг (WPF + EF Core)

Учебный проект для дисциплины "Программирование корпоративных систем".  
Реализует простое приложение для управления библиотекой с использованием WPF и Entity Framework Core.

## Функциональность

- CRUD операции для книг, авторов и жанров.
- Поиск книг по названию.
- Фильтрация книг по автору и жанру.
- Отображение количества книг в наличии.
- Каскадное удаление (при удалении автора/жанра удаляются все его книги).

## Технологии

- .NET 8
- WPF
- Entity Framework Core
- SQLite (файловая база `library.db`)
- CommunityToolkit.Mvvm

## Требования

- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- Никакого отдельного SQL Server не требуется — используется встроенная файловая база SQLite
- Редактор кода (VS Code, Visual Studio)

## Установка и запуск

1. **Установите зависимости**:
   - Установите [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0).

2. **Скачайте и откройте проект**:
   - Скачайте/распакуйте проект в любую папку, например `C:\development\book-library-management`.
   - Откройте папку проекта в Visual Studio / Rider / VS Code (или другом предпочитаемом IDE).

3. **Сборка из командной строки**:
   ```bash
   cd C:\development\book-library-management
   dotnet build
   ```

4. **Запуск приложения**:
   - Из IDE: запустите проект `LibraryManagement` как WPF-приложение (`F5` / `Start`).
   - Из командной строки:
     ```bash
     dotnet run
     ```

   При первом запуске приложение автоматически создаст файл базы данных `library.db` в рабочей папке (используется `Database.EnsureCreated()` с провайдером SQLite).
   Если в базе мало данных, приложение автоматически заполнит её демо-набором популярных **русскоязычных книг** (классика и современная литература). Все данные зашиты в код и **не требуют доступа к интернету**.

5. **Если база не создаётся / ошибки доступа к файлу**:
   - Убедитесь, что у вас есть права на запись в папку с приложением.
   - Если появляется окно с ошибкой, посмотрите текст исключения — обычно там указана причина (нет доступа к файлу, путь недоступен и т.п.).
